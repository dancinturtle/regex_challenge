In Django, we'll be using regular expressions for routing.

Regex can also be handy in many situations, and most languages support some form of regex. Some examples of uses for regex include text processing (think of chat bots or real language processing), data validation (like we have seen when working with form validation in Flask), and search queries. It's a great skill to have, and the more profficient you become the more it will start to feel like a super power!

But what is a regex, you might ask. A regex is simply a pattern we can match to any string. Find all the info you need to get started here: https://docs.python.org/2/library/re.html

In order to get started with regex in python, we must import the re library into our document:

import re

Regular expressions are preceeded by r, like so:

r"a"

This regex would match any string containing the letter a.

Let's take a look at some more complete code. We'll use this function to test some regular expressions. Don't try running the code yet, first, let's predict the output for the next few regular expressions.

  import re

  def get_matching_words(regex):
      results = []
      words = [
          "baby",
          "baseball",
          "denver",
          "facetious",
          "issue",
          "mattress",
          "obsessive",
          "paranoia",
          "rabble",
          "union",
          "volleyball",
      ]
      for word in words:
          if re.search(regex, word):
              results.append(word)

      return results

  my_expression = r"a"
  print get_matching_words(my_expression)
