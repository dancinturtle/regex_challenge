In Django, we'll be using regular expressions for routing.

Regex can also be handy in many situations, and most languages support some form of regex. Some examples of uses for regex include text processing (think of chat bots or real language processing), data validation (like we have seen when working with form validation in Flask), and search queries. It's a great skill to have, and the more profficient you become the more it will start to feel like a super power!

But what is a regex, you might ask. A regex is simply a pattern we can match to any string. Find all the info you need to get started here: https://docs.python.org/2/library/re.html

In order to get started with regex in python, we must import the re library into our document:

import re

Regular expressions are preceeded by r, like so:

r"a"

This regex would match any string containing the letter a.

Let's take a look at some more complete code. We'll use this function to test some regular expressions. Don't try running the code yet, first, let's predict the output for the next few regular expressions.

  import re

  def get_matching_words(regex):
      results = []
      words = [
          "baby",
          "baseball",
          "denver",
          "facetious",
          "issue",
          "mattress",
          "obsessive",
          "paranoia",
          "rabble",
          "union",
          "volleyball",
      ]
      for word in words:
          if re.search(regex, word):
              results.append(word)

      return results

  my_expression = r"a"
  print get_matching_words(my_expression)

Write regular expressions that will produce the following output, and use the above code to test your regex:

  1. ['issue', 'mattress', 'obsessive']
    2. Hint: Matches the literal "ss"

    r"(s)\1"

  2. ['baby']
    3. Hint: "." matches any character except for the new line, so this looks for a word with a "b", any character, then another "b"

    r"b.b"

  3. ['baby', 'baseball']
    4. Hint: "+" is a wildcard character meaning "one or more times", so this looks for a "b", one or more characters, then another "b". All of the wildcard characters (and the next few problems cover more) apply to the one character immediately in front of them (so then is any number of "."s, not any number of "b."s)

    r"b.+b"

  4. ['baby', 'baseball', 'rabble']
    5. Hint: "*" is like "+", but it means "zero or more times". This looks for a "b", any number of characters (including none), then another "b"

    r"b.*b"

  5. ['baby', 'rabble']
    6. Hint: "?" means that the preceeding character is matched 0 to 1 times, for example "."

    r"b.?b"

  6. ['facetious']
    7. Hint: This pattern matches a word that has all five vowels in order, regardless of how many letters are or are not between them

    r"a.*e.*i.*o.*u"

  7. ['issue', 'paranoia']
    8. Hint: Curly braces mean the pattern repeats that many times, so this is words that end with two vowels

    r"[aeiou]{2}\b"

  8. ['issue', 'paranoia', 'union']
    9. Hint: This pattern includes both "^" and "$", so it will only match full words. It looks for words consisting of one or more letters from "regular expression"

    r"(\w|^)(issue|paranoia|union)(\w|$)"

  9. ['baby', 'union']
    10. Hint: Annoyingly, "^" means two different things; inside a character class, it stands for "not", so this will match words that don't have any of the letters from "regex"

    r"^[^regex]*$"

  10. ['volleyball']
    11. Hint: This matches any word where the double-letter appears twice

    r"(\w)\1.*(\w)\1"
